# 요약 생성 프롬프트 지침

## 언어 및 톤
- 한국어로 작성
- 사실 기반, 간결체
- 원문에 없는 내용 추가 금지

## 카테고리 분류

아래 목록에서 가장 적합한 카테고리 1개를 선택:

| 카테고리 | 설명 |
|----------|------|
| AI | 인공지능, LLM, ML |
| DevTools | 개발 도구, SDK, API |
| Cloud | 클라우드, 인프라 |
| Security | 보안, 취약점 |
| Frontend | 프론트엔드, UI/UX |
| Backend | 백엔드, DB, 시스템 |
| Business | 인수합병, 펀딩, IPO |
| Open Source | 오픈소스 프로젝트 |
| HN | Hacker News 토론 |
| Misc | 위에 해당 안 되는 것 |

## 간단 요약 (불렛 3개)

- 글의 핵심 포인트 3가지를 각각 한 줄로 요약
- 각 불렛은 구체적 사실/수치를 포함
- "~했다", "~이다" 등 간결한 종결어미 사용

## 상세 정리 (20~30줄)

- 핵심 변경점 / 발표 내용으로 시작
- 글의 전개나 핵심 내용들을 메인 불렛 혹은 넘버 리스트 첫 레이어로 삼은 후,
- 불렛 및 넘버 리스트를 활용하여 내용을 체계적으로 전달
