### 1. OpenAI's Kevin Weil on the Future of Scientific Discovery

OpenAI VP of Science Kevin Weil이 a16z speedrun 창업자 Sam Shank와의 파이어사이드 챗에서 AI의 과학적 발견 가속화에 대해 논의했다.

**AI 역량의 빠른 진화**
- AI는 특정 과제에서 성공률 0~10%에서 60~80%까지 6~12개월 만에 도약한다
- 현재 프론티어 과학 응용 분야가 이 임계점에 도달해 있다
- "불가능 → 겨우 작동 → 탁월함"의 패턴이 반복적으로 관찰됨

**과학 연구의 미래상**
- AI 모델이 빠른 시뮬레이션 루프와 느린 실제 실험 루프를 동시에 운영하는 로봇 실험실 구상
- 인간 연구자의 물리적 제약(수면, 근무시간)을 제거하여 24시간 연속 실험 가능
- 연구자는 일상적 실험 작업 대신 더 높은 수준의 지적 작업에 집중

**업무 방식의 패러다임 전환**
- Weil 본인도 유휴 시간을 "낭비되는 컴퓨팅"으로 인식하게 되었다고 고백
- 여러 AI 에이전트가 백그라운드에서 병렬로 다른 프로젝트를 진행하는 방식으로 업무 수행
- 개인 생산성의 근본적 변화가 이미 시작됨

**스타트업 기회와 조언**
- 매달 아무도 예측하지 못한 새로운 AI 역량이 등장하여 "역사상 가장 비옥한 스타트업 기회"
- 단일 대형 모델에 의존하기보다 오케스트레이터 모델이 전문 소형 모델을 조율하는 앙상블 접근을 권장
- 많은 스타트업이 이 패턴을 간과하고 있다고 지적

### 2. Does AGENTS.md Actually Help Coding Agents?

ETH Zurich SRI Lab의 연구가 AGENTS.md, CLAUDE.md 같은 리포지토리 레벨 컨텍스트 파일이 코딩 에이전트 성능에 실제로 도움이 되는지 실증적으로 검증했다.

**실험 설계**
- Claude Code, Codex, Qwen Code를 대상으로 수백 개의 실제 GitHub 이슈에 대해 테스트
- 컨텍스트 파일 유무에 따른 성능 차이를 비교 분석

**LLM 자동 생성 파일의 역효과**
- 자동 생성된 컨텍스트 파일은 성공률을 0.5~2% 감소시킴
- 추론 비용은 20% 이상 증가
- 기존 문서에서 이미 확인 가능한 정보를 중복 재현하여 오히려 역효과 발생

**개발자 직접 작성 파일의 효과**
- 사람이 작성한 파일은 평균 4% 성능 향상을 보임
- 핵심 차이는 "신규성(novelty)"에 있음 — 코드만으로는 알 수 없는 비자명 정보 포함 여부가 결정적
- 기본값과 다른 도구 선택, 비관행적 설정 등 코드 외적 맥락이 중요

**지시 따르기의 역설**
- 에이전트는 컨텍스트 파일의 지시를 충실히 따라 더 많은 테스트를 실행하고 더 넓게 탐색함
- 그러나 활동량 증가가 더 나은 결과를 보장하지는 않음

**실무 권장사항**
- 기존 문서를 반복하기보다 정보 공백을 메우는 전략적 작성이 필요
- 컨텍스트 파일 하나당 약 20%의 추론 비용이 추가되므로 품질과 구체성이 핵심
- 연구는 Python 리포지토리에 한정되어 다른 언어에 대한 적용 가능성은 미확인

### 3. File System and CLI Is the Future Interface of AI-Native Products

John Jeong이 AI 네이티브 제품의 인터페이스로 파일시스템과 CLI가 최적이라는 주장을 펼쳤다.

**문제 제기: GUI의 한계**
- AI 태스크 매니저 aipm을 개발하면서 진짜 어려운 점이 모델이 아닌 인터페이스 설계였음을 발견
- Airstore, Ironclaw 등 다른 창업자들도 유사한 접근을 추구하고 있음을 관찰
- "GUI 상태는 종종 암묵적이다 — 숨겨진 토글, 분산된 설정, 쉽게 검사할 수 없는 로컬 상태"

**CLI·파일 시스템의 장점**
- CLI와 파일 기반 시스템은 명시적이고 검사 가능한 상태를 제공
- 마크다운 파일, 예측 가능한 폴더 구조, 플레인 텍스트 같은 안정적 구조가 핵심
- 이러한 안정적 구조 위에서 AI 에이전트가 필요한 통합 코드를 동적으로 생성 가능

**Obsidian 플러그인에서의 발견**
- 정교한 플러그인 생태계 대신, 안정적 파일 구조만으로 AI가 필요한 기능을 동적 생성할 수 있음을 확인
- 별도 플러그인 개발 없이도 파일 구조 자체가 확장 포인트가 됨

**핵심 논지: AI 네이티브의 재정의**
- "AI 네이티브"란 모델을 모던 UI로 감싸는 것이 아님
- 진정한 AI 네이티브 제품은 에이전트가 사람만큼 자연스럽게 조작할 수 있어야 함
- 파일시스템과 CLI는 구시대적 기술이 아니라, 이 목적에 가장 적합한 추상화
